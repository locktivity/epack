# Governance

epack is currently maintained by the Locktivity team.

## Decision Making

- Maintainers make final technical decisions on roadmap, architecture, and security posture.
- We prefer public discussion in issues and pull requests for decisions that affect users.
- Security-sensitive details may be handled privately until a fix is shipped.

## Maintainers

- Maintainers have merge and release authority.
- Maintainers are responsible for review quality, release safety, and security response.
- Maintainers may decline changes that increase unsafe defaults or widen attack surface without strong justification.

## Contribution Model

- All contributions follow `CONTRIBUTING.md`.
- Non-trivial behavior changes should be discussed in an issue before implementation.
- Breaking changes are allowed before `v1.0.0`, but must still be documented in `CHANGELOG.md`.

## Security Authority

- The security response process is defined in `SECURITY.md`.
- Vulnerability handling timelines and disclosure policy in `SECURITY.md` take precedence over normal issue/PR flow.

## Future Governance

As the project matures, governance can evolve to include:

- Additional maintainers from outside the founding team
- A lightweight steering group for major direction changes
- More formal voting/approval rules
